## Why

New contributors (and past-you) need a single place to understand what this repository is for and how Git, GitHub, and OpenSpec fit together. Right now `NOTES.md` only states that it is an OpenSpec project, which is not enough for someone learning these tools from scratch.

## What Changes

- Replace the stub in `NOTES.md` with a short, beginner-friendly overview of this repo.
- Explain in plain language: what Git is doing locally, what GitHub is for when you push, and how OpenSpec (and commands like `/opsx:propose`) support spec-driven changes in this project.

## Capabilities

### New Capabilities

- `beginner-repo-notes`: Onboarding documentation in `NOTES.md` that describes the repo’s purpose and how Git, GitHub, and OpenSpec apply here.
